# 全网采集引擎 v2.0 · 升级说明

编制：国嘉基业·LLM Wiki | 2026-06-28

## v1.0 → v2.0 六大升级

|||
|---|---|
|并发|单线程串行 → ThreadPoolExecutor 5线程并行|
|存储|JSON文件 → SQLite数据库（索引、去重、增量）|
|解析|CSS选择器规则 → 规则+LLM语义回退|
|JS渲染|不支持 → Playwright/Selenium探测，失败降级到移动API|
|去重|无 → SHA256哈希去重，同listing仅更新last_seen|
|增量|全量重采 → platform_state记录进度，只采新页|
|平台|20站 → 24站（新增4个）|
|7天过期|无 → 未再出现的listing自动标is_active=0|

## 数据库表结构
- listings: 核心供求数据（哈希去重）
- run_log: 每次采集运行日志
- platform_state: 每个平台采集进度状态

## v2.0 首轮运行结果
- 平台总数: 24 | 成功采集: 4站 | 总页面: 29页
- 总listing: 352条 | 去重后新增: 140条 | 并发加速: ~3.5x

## 每日运行方式
```
cd scrapers_v2/
./run.sh
python3 scraper.py --max-pages 5 --workers 8
```
