## 二、页面设计详述

### 2.1 首页（Dashboard）

设计目标：一屏展示核心信息，引导用户快速行动。

布局结构（自上而下）：
1. 搜索栏 — 品类/企业/地区全局搜索
2. 数据卡片行 — 在线供应数 × 采购需求数 × 今日匹配数（三列）
3. 快捷操作区 — 「发布供应」「发布需求」两个大按钮（2列网格）
4. AI智能匹配卡片 — 展示Top 3匹配结果，每条含：得分圆饼、标题、来源企业+报价、匹配维度标签
5. 行情快报卡片 — 4条重点品类当前价格+涨跌幅
6. 最新发布流 — 按时间倒序展示最近5条供需发布

关键交互：
- 点击匹配卡片 → 跳转匹配中心
- 点击行情卡片 → 跳转行情中心
- 快捷按钮 → 跳转发布页

### 2.2 发布供应页

设计目标：极简表单 + AI辅助，降低发布门槛。

表单字段：

| 字段 | 类型 | 必填 | AI辅助 |
|------|------|:---:|------|
| 图片上传 | 拍照/相册 | 否 | AI识别品类+颜色+预估指标 |
| 品类 | 下拉选择 | ✅ | 图片识别自动填充 |
| 形态 | 下拉选择 | ✅ | 图片识别自动填充 |
| 数量(吨) | 数字输入 | ✅ | - |
| 月可供量 | 数字输入 | 否 | - |
| 品质指标 | 文本输入 | 否 | AI辅助填写模板 |
| 报价(元/吨) | 数字输入 | ✅ | AI合理性检测提示 |
| 是否可议价 | 选择 | ✅ | - |
| 地区 | 省-市联动 | ✅ | - |
| 补充说明 | 多行文本 | 否 | - |

发布后流程：
点击「AI审核并发布」→ AI审核 → 发布成功 → 1.5秒后自动跳转「匹配中心」

### 2.3 发布需求页

设计目标：引导需求方精确描述要求，AI辅助匹配。

表单字段：

| 字段 | 类型 | 必填 | AI辅助 |
|------|------|:---:|------|
| 角色 | 选择 | ✅ | - |
| 目标品类 | 下拉选择 | ✅ | - |
| 技术要求 | 文本输入 | ✅ | AI推荐指标模板 |
| 月需求量 | 数字输入 | ✅ | - |
| 预算范围 | 文本输入 | 否 | AI合理性检测 |
| 应用场景 | 下拉选择 | ✅ | - |
| 地区 | 省-市联动 | ✅ | - |
| 认证要求 | 多选标签 | 否 | - |

AI辅助亮点：
- 用户选「PP再生颗粒」+「注塑托盘」→ AI自动推荐「建议熔指10-20，灰白色，冲击强度≥5kJ/m²」
- 用户输入预算5000-6500 → AI对比该品类近30天均价6200，提示「预算在合理范围内」

### 2.4 匹配中心

设计目标：清晰的匹配结果呈现，引导用户深入查看和行动。

筛选标签：全部匹配 | 强烈推荐(≥85%) | 我的供应匹配 | 我的需求匹配

匹配列表项：
```
┌──────────────────────────────────────────────┐
│ [92%] PET三色瓶砖 月需200吨                    │
│       安徽界首 · 再生PET工厂                    │
│       [强烈推荐] [5小时前]                      │
│                                            › │
└──────────────────────────────────────────────┘
```
得分圆饼颜色：≥85%绿色 | 70-84%蓝色 | 50-69%黄色

### 2.5 匹配详情浮层（Bottom Sheet）

设计目标：展示完整匹配分析，支持一键行动。

内容结构：
1. 头部 — 得分+标题+来源企业
2. 需求描述 — 灰底摘要
3. 六维雷达分析 — 品类/品质/数量/价格/区域/信誉，每项带进度条+百分比
4. 标签区 — 品类、供需特征标签
5. 行动按钮 — 「立即询价」（主按钮）+「收藏」（次按钮）

### 2.6 行情中心

设计目标：以最低认知成本展示市场价格全貌。

内容结构：
1. 价格指数趋势 — 14天迷你柱状图 + 文字说明
2. 热门品类行情表 — 8个重点品类当前价格+涨跌幅（红涨绿跌灰平）
3. 区域价差热力 — 同一品类不同区域价格对比（双卡片）

### 2.7 我的页面

设计目标：用户中心，管理发布、查看消息、编辑信息。

内容结构：
1. 用户卡片（绿色渐变背景）— 头像/名称/角色/认证状态 + 三维统计（发布/匹配/成交数）
2. 我的供应 — 最近的供应列表 + 「+发布」入口
3. 功能入口 — 企业名片、消息中心（红点提示）、数据看板(PRO标签)、设置
4. 关于 — 版本号、版权信息

## 三、核心交互流程

### 3.1 发布→匹配→撮合 主流程

```
用户打开小程序
    │
    ├── 首次使用 → 微信授权登录 → 选择角色 → 填写企业信息 → 进入首页
    │
    └── 已注册 → 直接进入首页
                  │
        ┌─────────┼─────────┐
        ↓         ↓         ↓
    [发布供应] [发布需求]  [浏览首页]
        │         │           │
        │    AI审核发布       │ 点击AI匹配卡片
        │         │           │
        └────┬────┘           │
             ↓                ↓
        ┌────────────────────────┐
        │     匹配中心           │
        │  查看匹配结果列表       │
        │  筛选：按匹配度/类型    │
        └────────┬───────────────┘
                 ↓
        ┌────────────────────────┐
        │  匹配详情浮层           │
        │  · 综合得分            │
        │  · 六维分析            │
        │  · 对方企业信息         │
        │  · 品类标签            │
        └────────┬───────────────┘
                 ↓
        ┌────────┼────────┐
        ↓        ↓        ↓
    [立即询价] [收藏]   [关闭]
        │        │
        ↓        ↓
   发送询价   加入收藏夹
   意向通知   后续可从「我的」
   对方       查看和跟进
        │
        ↓
   双方可在消息中心
   进一步沟通
        │
        ↓
   标记「已成交」
   → 反馈成交价
   → 数据反哺行情库
```

### 3.2 AI驱动节点

| 节点 | 说明 | 触发时机 |
|---|---|---|
| 图片识别 | 上传瓶砖/原料照片 → 识别品类/颜色/预估净瓶率 | 发布供应-上传图片 |
| 智能填表 | 根据品类推荐品质指标模板、合理报价区间 | 表单字段失焦时 |
| 价格检测 | 比对行情库，报价偏离均价±20%时给预警提示 | 提交发布前 |
| 智能匹配 | 发布后自动运行匹配算法，推送Top 10结果 | 发布成功后立即 |
| 匹配推送 | 新需求发布时，通知匹配度≥85%的供应方 | 实时监听 |
| 定价建议 | 根据品类+品质+区域+供需，推荐合理报价区间 | 数据看板查看时 |


## 四、设计规范

### 4.1 色彩系统

| 用途 | 色值 | 说明 |
|---|---|---|
| 主色 | #07C160 | 微信绿，品牌色 |
| 主色深 | #06AD56 | 导航栏/强调 |
| 信息蓝 | #1485EE | 链接/中等匹配 |
| 警告黄 | #FFC300 | 低匹配/注意 |
| 危险红 | #FA5151 | 涨价/错误/提醒 |
| 页面背景 | #F6F6F6 | 浅灰 |
| 卡片背景 | #FFFFFF | 白色 |
| 正文 | #191919 | 深黑 |
| 辅助文字 | #888888 | 灰色 |
| 浅文字 | #B2B2B2 | 浅灰 |
| 边框 | #E5E5E5 | 分割线 |


### 4.2 组件复用

- 匹配得分圆饼：48px → 64px（详情），三色（绿/蓝/黄）
- 标签Tag：10px字号，4px圆角，三种状态（绿底/蓝底/灰底）
- 卡片Card：12px圆角，0 2px 12px rgba(0,0,0,0.08)阴影
- 按钮：全宽，12px padding，8px圆角，单行
- 表单输入：全宽，8px圆角，#E5E5E5边框，focus变绿

### 4.3 页面尺寸

- 设计基准：iPhone 14 Pro（390 × 844px）
- 安全区：状态栏44px + 导航栏44px + 底部TabBar 64px = 内容区约692px
